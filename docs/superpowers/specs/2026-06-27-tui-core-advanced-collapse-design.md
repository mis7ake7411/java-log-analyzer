# TUI 核心表單簡化設計

## 背景
目前使用者介面一次展示的欄位較多，核心操作與進階設定混在同一層，容易讓第一次使用者難以快速找到最重要的輸入項目，也讓日常操作時的視覺負擔偏高。

## 目標
- 降低首屏操作複雜度，讓使用者能更快開始分析。
- 保留既有功能完整性，不移除任何既有分析能力。
- 讓進階設定在需要時才展開，平常不干擾核心操作。

## 非目標
- 不重新設計整個介面流程。
- 不移除任何既有欄位或分析能力。
- 不更動分析邏輯或輸出格式。

## 使用者故事
### 故事 1：快速開始分析
身為第一次使用者，我希望在進入介面後，能先看到最少而必要的欄位，這樣可以直接開始分析，不需要先理解全部選項。

驗收準則：
- 首屏可直接看到核心分析欄位與開始分析入口。
- 使用者不需要先展開進階內容，就可以完成一次基本分析。
- 核心欄位的順序固定且一致。

### 故事 2：保留進階控制
身為需要進階條件的使用者，我希望可以在同一頁展開進階設定，這樣在不切換畫面的前提下仍可完成完整設定。

驗收準則：
- 進階設定預設為收合狀態。
- 使用者可以在同一頁展開或收合進階設定。
- 展開或收合後，已輸入內容不會遺失。

### 故事 3：維持結果可讀性
身為使用者，我希望分析結果仍然維持清楚可讀，這樣我能快速判斷分析是否成功，以及輸出是否正常。

驗收準則：
- 分析完成後仍可清楚看到摘要資訊。
- 分割檔案資訊若存在，仍可被清楚辨識。
- 舊有結果閱讀方式不因表單簡化而失效。

## 核心欄位定義
主畫面只保留以下欄位：
- Log 目錄
- 目標資料夾
- 關鍵字
- 輸出格式
- 開始分析

## 進階區塊定義
進階區塊收納以下欄位：
- 開始日期
- 開始時間
- 結束日期
- 結束時間
- Logback XML
- Pattern
- 忽略大小寫

## 行為規則
- 進階區塊預設收合。
- 使用者可在同一頁展開或收合進階區塊。
- 使用者切換展開或收合時，不得清除已輸入內容。
- 預設狀態下，核心欄位應可直接完成一次基本分析。

## 成功準則
- 首次進入畫面時，使用者可在不捲動或最少捲動的情況下看到核心分析區。
- 核心欄位數量少於進階欄位，且視覺焦點明確。
- 展開進階區塊前後，任何已輸入資料都應保持不變。
- 現有分析流程與輸出結果維持可用。

## 驗收情境
1. 使用者只填入 Log 目錄、目標資料夾、關鍵字與輸出格式，即可完成分析。
2. 使用者展開進階區塊後填入時間區間與自訂格式，收合後再次展開，內容仍在。
3. 使用者不操作進階區塊時，主畫面仍可正常完成分析與顯示結果。

## 風險與假設
- 假設大多數使用者的日常任務以核心欄位為主，進階條件屬低頻需求。
- 假設進階區塊的收合不會阻礙既有工作流程的效率。
- 若後續驗證顯示核心欄位仍然過多，可再考慮進一步分組，但不在本次範圍內。
